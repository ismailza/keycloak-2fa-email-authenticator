# Authentication Flow Setup

Wire the Email OTP authenticator into a Keycloak authentication flow and configure realm email settings.

## Step 1 — Configure Realm Email (SMTP)

1. Login as **admin** to your Keycloak installation
2. Switch to your target realm
3. Navigate to **Realm Settings** → **Email** tab
4. Fill in your SMTP server details:

| Field               | Example                  |
| ------------------- | ------------------------ |
| SMTP Host           | `smtp.gmail.com`         |
| SMTP Port           | `587`                    |
| From Email          | `noreply@yourdomain.com` |
| Enable StarTLS      | ✅                       |
| Username / Password | your credentials         |

See [Email Provider Configuration](https://www.keycloak.org/docs/latest/server_admin/index.html#_email) for more details on these settings.

## Step 2 — Create the Authentication Flow

### a) Copy the Browser Flow

1. Navigate to **Authentication** → **Flows**
2. Select the **"Browser"** flow
3. Click **"Copy"** → name it `"Browser with Email OTP"`

### b) Add the Email OTP Step

In the `"Browser with Email OTP"` flow:

1. Select **"Browser with Email OTP Forms"**
2. Click **"Add execution"**
3. Select **"Email OTP"** (the provider from this project)
4. Click **"Add"**

### c) Set the Requirement

On the **"Email OTP"** row, set the requirement to:

- **REQUIRED** — always prompts for OTP on every login
- **CONDITIONAL** — only prompts based on a condition evaluator

### d) Bind the Flow

Under the **Bindings** tab:

- **Browser Flow** → select `"Browser with Email OTP"`
- Click **"Save"**

## Step 3 — Configure Email OTP Settings

Click the **⚙️ Settings** icon on the **"Email OTP"** row:

| Setting               | Description                                                                                      | Default |
| --------------------- | ------------------------------------------------------------------------------------------------ | ------- |
| **Code length**       | Number of OTP digits                                                                             | `6`     |
| **Time-to-live**      | Seconds before code expires                                                                      | `300`   |
| **Simulation mode**   | Log codes instead of sending emails                                                              | `false` |
| **Resend cooldown**   | Seconds before user can request a new code                                                       | `30`    |
| **Max code attempts** | Invalid attempts allowed before code is invalidated and a new one must be requested              | `5`     |
| **Skip setup**        | Users with an existing email skip the enrollment required action — useful for admin-enforced 2FA | `true`  |

:::tip Development tip — Simulation Mode
Set **Simulation mode** to `true` to test without sending real emails. Codes appear in the container logs:

```bash
podman logs -f keycloak-test | grep "SIMULATION MODE"
# ***** SIMULATION MODE ***** Email code send to test@example.com for user testuser is: 123456
```

:::
