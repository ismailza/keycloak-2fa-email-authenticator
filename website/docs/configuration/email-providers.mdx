# Email Providers

The authenticator supports four email providers. You choose one per authentication flow and can enable automatic fallback to Keycloak SMTP.

## Supported Providers

| Provider                    | Description                          | Requirements                        |
| --------------------------- | ------------------------------------ | ----------------------------------- |
| **Keycloak SMTP** (Default) | Uses Keycloak's built-in SMTP config | Realm SMTP settings configured      |
| **SendGrid**                | SendGrid cloud email service         | API key + verified sender           |
| **AWS SES**                 | Amazon Simple Email Service          | IAM credentials + verified identity |
| **Mailgun**                 | Mailgun email delivery               | API key + sending domain            |

## Configure Provider

1. Navigate to **Authentication** → **Flows**
2. Select your custom flow with the **Email OTP** authenticator
3. Click **⚙️** next to **Email OTP**
4. Set the provider and credentials (see options below)

### SendGrid Setup

1. Sign up at [sendgrid.com](https://sendgrid.com/)
2. **Settings** → **API Keys** → **Create API Key** (needs _Mail Send_ permission)
3. **Settings** → **Sender Authentication** — verify your sender email or domain
4. Enter the key and verified address in Keycloak

### AWS SES Setup

1. **IAM** → create a user with `AmazonSESFullAccess` (or `ses:SendEmail`). Save the access key pair.
2. **SES Console** → **Verified identities** → verify your sender email or domain (DNS records required for domain).
3. **Sandbox mode**: by default SES only sends to verified addresses. [Request production access](https://docs.aws.amazon.com/ses/latest/dg/request-production-access.html) to send to any recipient.
4. Choose a region close to your users — verified identities are region-specific.

### Mailgun Setup

1. Sign up at [mailgun.com](https://www.mailgun.com/)
2. **Settings** → **API Keys** → **Add new key** — copy the Private API key (starts with `key-`).
3. **Sending** → **Domains** → **Add New Domain** — use `mg.yourdomain.com` and complete DNS verification.
4. Region:
   - Account at `app.mailgun.com` → **US**
   - Account at `app.eu.mailgun.com` → **EU**

## Fallback Mechanism

When **Enable Fallback to Keycloak SMTP** is on:

- If the primary provider fails, the system automatically retries via Keycloak's realm SMTP
- Ensures delivery even when a 3rd-party service is temporarily unavailable
- Fallback events are logged for monitoring

:::tip Best practice
Always enable fallback in production. Configure realm SMTP as the safety net regardless of which primary provider you use.
:::
